# راهنمای اضافه کردن یک درس جدید به Kadoma Knowledge Base

این پوشه (`kie/data/`) همون بخش «Knowledge» ایه که دنبالش بودی —
جایی که فایل‌های درسی رو (هر کدوم حدود ۲۵۰۰ خط) پیست می‌کنی. الان
طوری آماده شده که برای **هر درسی** (نه فقط زیست) کار می‌کنه.

## قبل از شروع: هر فایل باید چه شکلی باشه؟

فایل `kie/data/TEMPLATE_subject.js` رو باز کن — این دقیقاً قالبیه که
باید پر کنی. یه نسخه از همون فرمتی که برای زیست استفاده شده، ولی
خالی و با توضیح فارسی کنار هر بخش.

اگه هوش مصنوعی دیگه‌ای (یا خودت) داری محتوای درس رو آماده می‌کنی،
کافیه بهش بگی: **«این فایل رو دقیقاً با همین ساختار JSON پر کن»**
و همین `TEMPLATE_subject.js` رو بهش نشون بدی.

## ۳ مرحله برای اضافه کردن یک درس

### مرحله ۱ — فایل رو بساز
یه کپی از `TEMPLATE_subject.js` بگیر، توی همین پوشه (`kie/data/`)
بذارش، و اسمش رو بر اساس درس عوض کن. مثلاً:

```
kie/data/chemistry11.js
kie/data/physics10.js
kie/data/math12.js
```

داخل فایل، `SUBJECT_KB` رو با محتوای واقعی همون درس پر کن (فصل‌ها،
مبحث‌ها، توضیح‌ها). هر چند فصل که کتاب داره اضافه کن، محدودیتی نیست.

### مرحله ۲ — یک خط import اضافه کن
بالای فایل `kie/data/index.js`، کنار importهای زیست، این خط رو
اضافه کن (اسم فایل و اسم متغیر رو مطابق درس خودت عوض کن):

```js
import { SUBJECT_KB as CHEMISTRY_11_KB } from './chemistry11.js';
```

### مرحله ۳ — یک خط به لیست SUBJECT_FILES اضافه کن
توی همون فایل `index.js`، دنبال آرایه‌ی `SUBJECT_FILES` بگرد (یه
کامنت بزرگ بالاش نوشته «SUBJECT REGISTRY» — پیدا کردنش راحته) و یه
خط جدید مثل خط‌های زیست بهش اضافه کن:

```js
const SUBJECT_FILES = [
  { kb: BIOLOGY_10_KB, subjectName: 'زیست‌شناسی (پایه دهم)', idPrefix: 'zist10' },
  { kb: BIOLOGY_11_KB, subjectName: 'زیست‌شناسی (پایه یازدهم)', idPrefix: 'zist11' },
  { kb: BIOLOGY_12_KB, subjectName: 'زیست‌شناسی (پایه دوازدهم)', idPrefix: 'zist12' },
  { kb: CHEMISTRY_11_KB, subjectName: 'شیمی (پایه یازدهم)', idPrefix: 'chem11' }  // ⬅️ خط جدید
];
```

**نکته‌ی مهم درباره‌ی `idPrefix`:** باید کوتاه و برای هر درس/پایه
یکتا باشه (مثل `chem11`, `phys10`, `math12`) — این پیشوند جلوی
تداخل شماره‌ی فصل‌های درس‌های مختلف رو می‌گیره.

## همین. کار دیگه‌ای لازم نیست.

هیچ فایل دیگه‌ای (نه Engine، نه Adapter، نه هیچ‌جای دیگه) نیاز به
تغییر نداره — این ۳ خط کافیه.

## چطور روی گیت‌هاب آپلود کنم؟

۱. فایل جدیدت (مثلاً `chemistry11.js`) رو توی همون مسیر
`kie/data/` روی گیت‌هابت آپلود کن (کنار `biology10.js`).
۲. فایل `index.js` رو (که الان داخلش این دو خط رو اضافه کردی) هم
جایگزین نسخه‌ی قبلیش کن.
۳. تمام — بقیه‌ی پروژه دست‌نخورده می‌مونه.

## چک کردن اینکه درست کار کرده (اختیاری)

اگه می‌خوای مطمئن بشی درس جدیدت درست ثبت شده:

```js
import { attachKnowledgeBase } from './kie/data/index.js';
const kb = attachKnowledgeBase(stage3);
console.log(kb.ingestedChapterIds); // باید فصل‌های درس جدیدت رو هم توش ببینی
```
